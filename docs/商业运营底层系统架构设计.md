# 商业运营底层系统架构（1 人公司极简版）

> 目标：只做三件事——看清钱从哪来（营销）、看清货好不好（产品）、看清人有没有变强（学员）。  
> 适用：一人公司，从 0 到 1，先跑通再扩张。  
> 技术栈：**Rust + Vue + PostgreSQL**。

---

## 1. 只保留一个经营目标

### 北极星指标（每周看一次）

**有效营收 = 收款金额 × 完课率 × 作业达标率**

为什么用这个：

- 只看收款会“卖得动但交付差”；
- 加上完课率和达标率，才能保证长期复购与口碑。

---

## 2. 一人公司只跑 3 个闭环

## A. 钱从哪来（营销闭环）

内容发布 → 私聊/表单线索 → 体验课/咨询 → 付费

每周只看 4 个数：

1. 线索数
2. 咨询数
3. 成交数
4. 成交率（成交数/咨询数）

> 原则：只保留 1~2 个主渠道，不做“全平台分散”。

## B. 货好不好（产品闭环）

开课 → 作业提交 → 批改反馈 → 毕业项目

每周只看 4 个数：

1. 到课率
2. 作业提交率
3. 作业达标率
4. 退款率

> 原则：每周必须有“可验证交付物”，比如代码、报告、项目演示。

## C. 人有没有变强（学员闭环）

入学测评 → 阶段测评 → 毕业测评

每期只看 3 个数：

1. 学员能力提升率
2. 毕业达标率
3. 可展示案例数

> 原则：没有前后测，就不要说“学员变强了”。

---

## 3. 极简业务流程（你一个人也能执行）

## 3.1 招生 SOP

1. 每周固定发布 3 条内容（痛点/案例/方法）
2. 引导到同一个报名表单
3. 每周 1 场体验课或咨询
4. 成交后自动发送入学说明

## 3.2 交付 SOP

1. 开课前：发布课表 + 作业标准
2. 开课中：每周一次直播 + 一次集中答疑
3. 开课后：每周批改一次作业并给统一反馈模板
4. 毕业时：项目验收 + 结果归档

## 3.3 复盘 SOP（每周 1 小时）

- 渠道复盘：哪条内容带来的成交最多
- 交付复盘：哪周掉队最多、为什么
- 下周动作：只定 3 条改进动作

---

## 4. 技术架构（最小可用）

## 4.1 你现在只需要 1 个后台 + 1 个数据库

### 前端（Vue）

- 一个管理后台，3 个页面就够：
  1. 线索与订单
  2. 班级与作业
  3. 经营看板

### 后端（Rust）

- 一个 API 服务（建议 Axum）：
  - 线索接口
  - 订单接口
  - 作业与测评接口
  - 看板聚合接口

### 数据库（PostgreSQL）

- 一库到底，先不拆服务，不上消息队列。

---

## 4.2 最小数据表（9 张就够）

1. `leads`（线索）
2. `consultations`（咨询记录）
3. `orders`（订单）
4. `cohorts`（班级）
5. `students`（学员）
6. `assignments`（作业）
7. `submissions`（作业提交）
8. `assessments`（测评）
9. `weekly_metrics`（周指标快照）

---

## 4.3 三个核心 API（先做这三个）

1. `POST /api/leads`：收集线索
2. `POST /api/submissions`：提交作业
3. `GET /api/dashboard/weekly`：查看每周经营数据

只要这三个接口跑通，你就能从“凭感觉”转向“看数据做决策”。

---

## 5. 看板只保留一页（Weekly Dashboard）

每周固定展示 10 个指标：

- 线索数、咨询数、成交数、成交率
- 收款金额、退款率
- 到课率、作业提交率、作业达标率
- 学员能力提升率

> 原则：每多一个指标，都是维护成本。先把这 10 个数做准。

---

## 6. 30 天落地计划（单人可执行）

### 第 1 周：打底

- 定义产品（你卖什么）
- 定义毕业标准（学员变强的证据）
- 建 PostgreSQL 表结构

### 第 2 周：上线最小系统

- Rust 后端完成 3 个核心 API
- Vue 后台完成 3 个页面
- 跑通“线索→成交→入学”流程

### 第 3 周：跑第一期交付

- 按周交付课程与作业
- 记录提交率、达标率、掉队名单
- 完成第一次周复盘

### 第 4 周：优化并固化

- 下线无效渠道
- 优化最差一环（通常是咨询转化或作业提交）
- 固化 SOP 与指标定义

---

## 7. 你现在最该做的 3 件事

1. 先写清楚“毕业达标标准”（人有没有变强）。
2. 再写清楚“每周交付物标准”（货好不好）。
3. 最后只选 1~2 个获客渠道（钱从哪来）。

只要这三件事清晰，一人公司也能把业务跑成“可复制系统”。
